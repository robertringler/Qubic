| Base Dark | `#002b36` | Primary background |

All designs must maintain a dark-theme aesthetic:

## Integration Workflow

### Step 1: Figma Project Setup

1. **Create Project**: Name it `QuASIM_QuNimbus_Marketing`
2. **Organize Pages**:
   - Brand Assets (logo, colors, typography)
   - Marketing Pages (landing, features, about)
   - Pitch Deck
   - Social Media Templates
   - Diagrams (architecture, flow charts)
   - Illustrations (quantum elements, icons)

3. **Setup Components**:
   - Button styles (primary, secondary, text)
   - Card components (feature, metric, testimonial)
   - Icons (create or use Feather/Heroicons)
   - Typography styles (H1-H6, body, caption)

### Step 2: Design Creation

1. **Reference Existing Assets**: Check `docs/assets/` for current logos and branding
2. **Follow Brand Guidelines**: Use specified color palette and typography
3. **Create Frames**: One frame per deliverable (landing page, pitch slide, etc.)
4. **Use Auto Layout**: For responsive components
5. **Name Layers Clearly**: For easy export identification

### Step 3: Tagging and Documentation

Tag each frame with relevant metadata:

```
Frame Name: Landing_Hero_Desktop
Tags: #QuASIM #QuNimbus #QuantumSimulation #MarketIntelligence
Links: 
  - README.md#executive-summary
  - docs/technical/architecture.md
  - docs/valuation_dashboard.html
```

### Step 4: Export Assets

#### Via Figma Interface

1. **Select Frame/Layer**
2. **Export Settings** (right panel):
   - Format: SVG (for logos, icons, diagrams)
   - Format: PNG (for screenshots, social media)
   - Scale: 1x (actual size), 2x (retina)
3. **Export**
4. **Organize** into appropriate directories

#### Via Figma API (Automated)

Use the Figma REST API to automate exports:

```bash
# Example: Export all frames from a file
curl -H "X-Figma-Token: YOUR_TOKEN" \
  "https://api.figma.com/v1/files/FILE_KEY/images" \
  -o figma_exports.json
```

See [Figma API Documentation](https://www.figma.com/developers/api) for details.

### Step 5: Commit to Repository

```bash
# Navigate to repository root
cd /path/to/QuASIM

# Stage assets
git add docs/assets/marketing/*.svg
git add docs/assets/marketing/*.png
git add docs/assets/social/*.png
git add figma/

# Commit with descriptive message
git commit -m "docs(assets): add QuASIM×QuNimbus Figma visuals and marketing suite"

# Push to branch
git push origin marketing/figma-assets-v1
```

## Export Specifications

### SVG (Vector Graphics)

**Use for**: Logos, icons, diagrams, illustrations

**Settings**:

- Format: SVG
- Include: "id" attributes
- Outline text: Optional (keep editable if possible)
- Simplify paths: Yes
- File size optimization: Enabled

### PNG (Raster Graphics)

**Use for**: Screenshots, social media, photographic elements

**Settings**:

- Format: PNG
- Scale: 1x (standard), 2x (retina)
- Background: Transparent (if applicable)
- Color space: sRGB

## Asset Management

### Naming Conventions

Use clear, descriptive names:

```
{category}_{type}_{variant}_{size}.{ext}

Examples:
- landing_hero_desktop_1920x1080.png
- feature_gpu_acceleration_icon.svg
- social_twitter_card_v1.png
- diagram_architecture_overview.svg
- badge_cmmc_level2.svg
```

### Version Control

- **Figma**: Use branching and version history
- **Git**: Commit exported assets with version tags
- **Changelog**: Document major changes in `CHANGELOG.md`

### File Size Optimization

- **SVG**: Use SVGO or similar tools to optimize
- **PNG**: Use ImageOptim, TinyPNG, or similar
- **Target**: < 500KB for web assets
- **Large Files**: Store in Git LFS if needed

## Best Practices

### Design Best Practices

1. **Consistency**: Use components and styles from design system
2. **Accessibility**: Maintain WCAG AA contrast ratios (4.5:1 for body text)
3. **Responsiveness**: Design for multiple breakpoints
4. **Performance**: Optimize assets for web (file size, format)
5. **Branding**: Always follow brand guidelines

### Collaboration Best Practices

1. **Comments**: Use Figma comments for feedback and questions
2. **Prototyping**: Create interactive prototypes for user flows
3. **Handoff**: Use Figma Dev Mode for developer handoff
4. **Reviews**: Schedule regular design reviews with stakeholders
5. **Documentation**: Keep this guide updated with new patterns

### Export Best Practices

1. **Batch Export**: Export related assets together
2. **Check Output**: Verify exported files before committing
3. **Organize**: Place files in correct directories immediately
4. **Name Clearly**: Follow naming conventions
5. **Document**: Update README files in asset directories
